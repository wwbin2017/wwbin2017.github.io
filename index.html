c++容易混淆的概念

发表于 2019-05-01

&（左值引用） 和 && 右值引用

&（左值引用）：需要注意的是生命周期，
c++ 中的左值就像指针，它可以捕获实实在在的实体，但是我们要注意被捕获值的生命周期。不要随便把生命周期和栈同步的实体传给了它；

&& 右值引用
右值其实也是指针，但是它功能是专门捕获匿名的实体（可以理解为编译产生的中间变量）。同时我们要注意，右值在定义时捕获了实体以后，右值的名字就变成了被捕获的实体。

类类型

类的声明：仅声明类暂时不定义它

这种声明被称作 “前向声明”，在它声明之后定义之前是一个不完全类型（incomplete type），即我们知道是个类型，但是不知道内部情况

不完全类型只能在非常有限的情况下使用：指针或者引用，参数或者返回值（都不会创建类型对象）

只有类被定义之后，数据成员才能才能被申明成这种类型，所以一个类的成员类型不能是自己（如果可以的话，就会无限循环了）

static

  1. 静态成员的定义和声明要加个关键static。静态成员可以通过双冒号来使用，即<类名>::<静态成员名>。
  2. 初始化静态成员变量要在类的外面进行。初始化的格式如下：数据类型 类名::静态成员变量名 = 初值；

g++ 编译参数说明

发表于 2019-03-24

g++ -I -L -l

-I 头文件路径 查找路径
-L 库文件路径 查找路径
-l 库链接 需要用到的库
就拿数学库来说，他的库名是m，他的库文件名是libm.so，很容易看出，把库文件名的头lib和尾.so去掉就是库名了

/lib和/usr/lib和/usr/local/lib里的库直接用-l参数就能链接了，但如果库文件没放在这三个目录里，而是放在其他目录里，这时我们只用-l参数的话，链接还是会出错，出错信息大概是：“/usr/bin/ld: cannot find-lxxx”，也就是链接程序ld在那3个目录里找不到

LD_LIBRARY_PATH等环境变量置顶的路进去查找，如果还找不到，那么编译程序提示找不到库。

动态链接库（通常以.so结尾）和静态链接库（通常以.a结尾）

LIBRARY_PATH环境变量：指定程序静态链接库文件搜索路径
LD_LIBRARY_PATH环境变量：指定程序动态链接库文件搜索路径

Trapping Rain Water

发表于 2018-12-08

LeetCode 题目：Trapping Rain Water 难度：hard

Given n non-negative integers representing an elevation map where the width of each bar is 1, compute how much water it is able to trap after raining.
Example:

Input: [0,1,0,2,1,0,1,3,2,1,2,1]
Output: 6

注解：对问题进行分析后，也即是如果看起来是U型的话，才可能蓄水。因此，只要遍历找到U型就行了。最后结果只超越了23.18% ，应该不是最优解，最多算是暴力解法。感觉最近迷恋暴力解法了，如果可以通过的话。看了讨论果然有更好的解法，见最后的说明。

更优解法-solution-with-2-loops-Space-Complexity-O(1)-beats-97)

基本思路，从两边遍历

  • 左到右：找到左边为U型水槽，并且左边边界比右边边界低
  • 右到左：找到右边为U型水槽，并且左边边界比右边边界高

比较关键的是两种遍历是否有重叠，当然已经成功了，说明没有重合。

比较受欢迎的解法

基本思路差不多，都是找到最低位置，以最低位置为标准，直接计算。

First Missing Positive

发表于 2018-12-07

LeetCode 题目：First Missing Positive 难度：hard

Given an unsorted integer array, find the smallest missing positive integer.

Example 1:

Input: [1,2,0]
Output: 3
Example 2:

Input: [3,4,-1,1]
Output: 2
Example 3:

Input: [7,8,9,11,12]
Output: 1
Note:

Your algorithm should run in O(n) time and uses constant extra space.

初看之下没想到好的解法，后面想到一种比较巧妙的解法。问题的描述是找到最小的正整数。换句话说，数字的出现是否连续，我们可以换成位运算，如果1-100出现，对应的第一位为1，如果100-200出现，对应的第二位为1，如此。最后，我们对每一个位判断是否位1，不为1，则就是缺失的值，时间和空间复杂度满足要求。或许有更优的解法。
